## 0、AggregateEvidence 和 DecisionNode运行框架
Agent 分析输出文件（agent_output/）：你的担忧是对的，但这是设计行为而非污染。AggregateEvidence 和 DecisionNode
节点会在 prompt 里明确引用上一个交易日的判断框架作为对比基准（"从昨日震荡偏多升级为偏多"），这是 GoT 框架里的 delta
分析，不是信息泄露。如果你认为每日判断应完全独立（不参考昨日结论），那需要从 prompt 模板里删除"与前一日对比"这条指令

## 一、批判节点与回流

| | 说明 |
|--|------|
| **现状** | `CriticNode` 审 `aggregate` + `decision`；若 **`HumanInput_{D}.json`** 非空，prompt 要求核对人类约束与决策一致性。允许回流时最多回到 `AggregateEvidence` 一次。Agent 读盘模式常为 `skip_revision_loop`，不自动多轮批判。 |
| **缺口** | 批判不能指向「只重跑某一个上游节点」（如仅 `MacroNews`）；回流只能「再归并一遍」，输入 JSON 不变则根因难自动修，常需人工改快照后重跑。 |
| **方向** | 批判输出可解析的 `target_nodes`；图路由支持按节点重算后再归并；或「标节点 + 人工确认再触发」半自动；约定可覆盖的 JSON 与审计。 |

## 四、新闻资讯的基本面量化与海量数据处理（行业参考）

本节记录 **量化金融侧常见做法**（非本仓库已实现能力），供后续将「资讯 MCP → 结构化因子 / 宽表」升级时对照；与当前 MVP 中 **`MacroNews.json` 等「多行日期+标题」字符串快照** 的关系见文末一句收束。

### 4.1 新闻如何「基本面化」

新闻不直接等于会计科目，业界通常先做 **结构化 + 映射到经济/公司状态变量**，再进因子或模型：

- **实体链接（NER + 消歧）**：公司、行业、国家、产品等 → 对齐 **统一证券标识**（Permno、FIGI、Wind 代码等）及 **行业/供应链树**，以便与财报、估值、宏观序列对齐。  
- **事件类型分类**：盈利预警、监管、订单、并购、分红回购、政策、地缘等 → 离散 **事件标签**（taxonomy 思路可参考 RavenPack、Capital IQ Key Developments 等商业库）。  
- **情感与极性**：句级/文档级 **sentiment score**，再按 **证券 × 日** 聚合（加权平均、极值、尾部占比等）。

与「基本面」拼接的常见接口：

| 接口 | 做法 |
|------|------|
| **横截面因子** | 某日每只股票一个 **新闻情绪、事件计数、不确定性** 等，与价值、质量、动量等一并进多因子或风险模型。 |
| **时间序列 / Nowcasting** | 对宏观或行业将新闻聚合成 **日/周活动指数**（文本版「经济意外」），用于 **现报 GDP、消费、政策松紧** 等。 |
| **事件研究** | 公告级、并购级等 → **异常收益（CAR）**、波动跳跃，或 **结构断点** 与分段基本面估计。 |

共识：**新闻更擅长边际变化、尾部风险与预期差**；常与 **财报、高频量价、供应链数据** 并用，而非单独替代利润表。

**前沿简提**：大模型做 **信息抽取与可验证三元组**；区分事实与观点以降低假相关；电话会/路演等多模态与新闻一致化（合规要求更高）。

### 4.2 大量原文如何处理（数据工程惯常路径）

典型是 **流水线**，而非把全文直接塞进交易信号层：

1. **采集与入库**：实时流 + 批量补数；**原始层（raw）** 尽量追加、少改写，保留 URL、时间戳、来源、原文或合规片段，便于审计与 NLP 重跑。  
2. **清洗与去重**：跨源转载用 SimHash/MinHash、URL 归一、标题相似度等 **去重或主从**；规则 + 轻量模型去垃圾；**时间对齐** 与 **Point-in-Time（PIT）** 避免前视偏差。  
3. **分层存储**：Bronze / Silver / Gold（或等价）：原文 → 清洗文本 → **实体/事件/情感表**（列式 Parquet、Iceberg、ClickHouse 等）；**聚合层** 按 **证券-日、行业-日、宏观主题-日** 预聚合供回测与低延迟读取。  
4. **规模与成本**：对非核心股票可降采样，对 **指数成分与持仓池** 全量；异步重算（先词典版上线、再离线大模型精修 **Silver**）；API **单次条数上限**（如 `search_news` **size≤20**）在工程上靠 **分页、多 query、多源合并** 逼近「窗内全量」，再写入宽表或文档库。  
5. **与模型衔接**：标签稀疏处用 **半监督/主动学习**；因子层做 IC、分层、换手与容量、与风格因子正交化；**合规** 下常只存摘要+元数据或采购持牌数据商。

### 4.3 与本项目 MVP 的关系（收束）

当前快照把多命中 **拼进少数大字符串字段**，且每条 **仅存日期+标题、不含正文片段**（见 [`MCP_资讯_取数.md`](./src/got_mvp/prompt_md/取数/MCP_资讯_取数.md)），可视为 **轻量索引 / Gold 层前的标题目录** 或 Agent 输入；若产品化升级，下一步通常是 **解析为行表（证券-时间-事件-分数）** 或 **另建 raw 层保留 URL+全文** 再进因子库或图状态，并与 **§二 MCP 资讯** 的「多 query 去重」方向一致。
